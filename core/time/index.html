    accurate_sleep :: proc "contextless" (d: Duration) {â€¦}
     

    Accurate sleep borrowed from: https://blat-blatnik.github.io/computerBear/making-accurate-sleep-function/

    Accuracy seems to be pretty good out of the box on Linux, to within around 4Âµs worst case. On Windows it depends but is comparable with regular sleep in the worst case. To get the same kind of accuracy as on Linux, have your program call win32.time_begin_period(1) to tell Windows to use a more accurate timer for your process.
